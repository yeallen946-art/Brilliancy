# Brilliancy 微信小程序(大陆主入口)

PRD §12 的落地骨架。原生小程序(WXML/WXSS/JS),零构建链、零第三方依赖。
架构与 iOS 端同源(TECH_SPEC §11):**客户端零棋类引擎、零 LLM 调用**,
一切内容来自管线预计算的每日 JSON。

## 跑起来

1. 微信开发者工具 → 导入项目 → 选 `miniprogram/` 目录。
2. `project.config.json` 的 `appid` 是占位(`touristappid` 可用游客模式预览);
   正式开发改为公司小程序的 appid。
3. 开发期勾选「不校验合法域名」(详情 → 本地设置),即可从 GitHub Pages 拉内容。
   **上线前**改 `utils/content.js` 的 `BASE_URL` 为境内备案域名(PRD §12.5),
   并在小程序后台配置 request 合法域名。

## 关键设计(为什么不需要引擎)

- **合法着法 = `legal_evals` 的键**:管线对每个猜测点穷举了全部合法着法,
  点选走子的目标格直接由键前缀过滤得出。
- **走子应用纯机械**(`utils/chess.js`):易位 = 王横移两格则搬车;吃过路兵 =
  兵斜走到空格则移除过路兵;升变 = UCI 第 5 字符。无任何走法生成/合法性逻辑,
  因此不触碰 CLAUDE.md 硬规则 #5 的正确性测试面。
- **SAN 全部预计算**(`san` / `refutation_san`),客户端只显示。
- **计分**(`utils/scoring.js`)是 ScoringConfig 的纯函数移植——常量以
  TECH_SPEC §3.2 为单一事实来源,调参两端同步。
- **讲解**直接读 `annotation_zh` / `alt_annotations_zh`;长尾错着用中文模板
  拼引擎数字(GuessExplainer 的移植),不产生任何自由棋评(硬规则 #1)。
