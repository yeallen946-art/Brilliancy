# PRD — Brilliancy

iOS 国际象棋"猜大师着法"训练应用 · 产品需求文档 v1.0
2026-06-07 · Owner: Jerry

---

## 1. 一句话定位

每天 5 分钟,跟大师下一盘名局:猜出大师的下一步,AI 用人话告诉你「为什么你的着法不如他的」。

## 2. 背景与机会

- 国际象棋手游市场 2025 年约 3.7 亿美元/年,头部(Chess.com)垄断,但细分训练品类有独立 app 的生存空间。
- "Guess the move" 玩法已被验证(ChessTempo、Forward Chess 免费提供,App Store 已有同类付费 app),但**全市场没有一家解释"为什么你的着法更差"**——引擎只给 ±0.3 这种冰冷数字。
- 差异化 = AI 自然语言讲解(锚定引擎输出)+ 进步追踪 + 每日挑战的仪式感。

## 3. 目标用户

**核心画像:改善型玩家(Improver)**

- 等级分 800–2000(USCF/Chess.com),知道"打谱学名局"有用但觉得枯燥
- 看 GothamChess / Hikaru,做战术题,但棋力停滞
- 痛点:打谱没反馈、不知道自己学到没有、坚持不下来

**种子用户:** Jerry 的儿子(USCF 1700),即代表性用户,承担可用性评审(见 §7)。

**非目标用户:** 纯新手(不懂规则)、2200+(需要的深度本产品不提供)。

## 4. 产品结构(混合型)

```
免费漏斗:每日挑战(拉新、留存、传播)
    ↓
付费核心:训练库(题库 + AI 讲解 + 进步追踪)→ 订阅
```

### 4.1 每日挑战(免费)

- 每天全球用户猜**同一盘名局**(精选 20–30 个关键着法点)
- 完成后出总分 + 可分享成绩卡(类 Wordle 表情格,不剧透具体着法)
- 连胜 streak,断签提示
- 历史每日挑战归档 → 付费解锁

### 4.2 训练库(付费核心)

- 按棋手分包:Tal 攻杀包、Morphy 包、Fischer 包、Carlsen 包……
- 按主题分包:王翼进攻、残局技术、防守反击……
- 每局含 AI 讲解(见 §6)
- MVP 内容量:**精讲 50 局**(每局全程可猜 + 全程讲解),宁缺毋滥;后续每月增补

### 4.3 进步追踪(付费)

- **棋感评分(Guess Rating)**:类 Elo,随每次猜测更新,有历史曲线
- 弱点分析:按局面类型(战术/局面/残局/开局)分维度统计命中率
- 评分曲线刻意调得前期略宽容,让用户两周内看到上涨(续订动机)

## 5. 核心玩法定义

**猜着法循环:**

1. 展示局面(用户执大师一方),给出对局背景(选手、年份、赛事)
2. 用户在棋盘上走出自己的猜测
3. 评分(三档):
   - **命中**(= 大师实着):满分
   - **同等好棋**(引擎评估差 ≤ 0.3 且不劣于实着):高分,并提示"你的着法同样成立,大师选择了另一条路"
   - **偏差**:按引擎评估差扣分;严重失误(≥ 1.5)标红
4. 展示大师实着 + AI 讲解(为什么这步好 / 为什么你的那步不如它)
5. 自动推进到下一个猜测点(对手着法快速播放)

**关键体验原则:**

- 不是每步都猜——开局照搬阶段快进,只在"有内容可讲"的着法点停下(每局 15–30 个点,内容流水线标注)
- 单局时长控制在 5–10 分钟
- 猜错不羞辱:讲解的语气是教练,不是判官

**猜测点节奏:两个流派与我们的选择(2026-06-11 定):**

业界同类产品分两派。**流派一"整局全猜"**(Chess Tempo Guess The Move、Pandolfini《Solitaire Chess》):选边后你方每步都猜,显然着法不跳过而是低分值(Pandolfini 用分值权重解决无聊步),单局 30–40 个点、20 分钟以上,桌面深度学习工具的节奏。**流派二"精华剪辑"**:少量高戏剧性猜测点 + 中间自动走子,单局 5–10 分钟,移动端日活节奏。

**我们选流派二**(与本节体验原则一致),落地规则:

- 机器初筛(跳书着、跳强制回吃)之后,**自动剔除"唯一着法"**——最佳与次佳差距 ≥ 2.5 兵且非将杀类的点没有猜的乐趣(将杀/弃子序列豁免:那正是戏剧本身,连续猜测点如 Réti 的三连杀是对的);
- **人工终审有最后决定权**:`content/curation/guesspoint_overrides.json`(进 git)按局增删猜测点,机器提案、人来剪辑;
- 流派一可作为训练库的付费"整局模式"进 backlog(Chess Tempo 验证过核心棋迷吃这套);显然着法低分值加权是扩量时的备选方案。

## 6. AI 讲解(核心差异化)

**定义:** 对每个猜测点,预生成自然语言讲解,覆盖:实着的意图、主要威胁/计划、以及针对最常见的 2–3 种错误猜测的"为什么不行"。

**硬性规则(产品层面):**

1. 讲解**必须锚定引擎输出**(Stockfish 评估值 + 主变),LLM 只负责把引擎变着翻译成人话,禁止自由发挥棋理
2. **预生成、不实时**:内容流水线离线批量生成,控制成本、保证质量(技术细节见 TECH_SPEC)
3. 语言水平对齐 1200–1800 读者:讲计划和威胁,不堆变着

**质检流程(双层):**

- 正确性 → 引擎自动校验(讲解中的着法必须在引擎 top moves / 主变内,评估描述与数值一致)
- 有用性 → 代表性用户评审(1700 棋手通读:读得懂吗?讲到点子上了吗?像人话吗?)抽查比例 MVP 期 100%,稳定后 ≥ 20%

## 7. 变现

**模式:Freemium 订阅 + 买断**

| 层级 | 价格 | 包含 |
|---|---|---|
| 免费 | $0 | 每日挑战、3 局体验精讲局、引擎对错判定 |
| 订阅 | $4.99/月 或 $29.99/年 | 全部题库、AI 讲解、棋感评分与弱点分析、每日挑战归档 |
| 买断 | $49.99 | 同订阅(棋类用户吃 Chessable 式买断逻辑) |

**原则:** 付费的是**讲解和进步追踪**,不是"更多棋局"(棋谱是公版的,量不值钱)。

- 苹果小企业计划:抽成 15%
- 试用:订阅 7 天免费试用
- 转化触点:每日挑战结束后展示"想知道为什么第 14 步你错了?"→ 引导订阅

## 8. 留存与增长

**留存:**

- 每日挑战 + streak(核心日活引擎)
- 每日推送一条有故事性的标题:"今天:Fischer 13 岁的世纪之局(1956)"
- 棋感评分涨跌反馈

**增长(获客内建在产品里):**

- 分享卡:每日挑战成绩卡,一键分享(iMessage / 社交),不剧透
- ASO 关键词:chess training, guess the move, chess puzzles, master games
- 冷启动渠道:r/chess、Chess.com 论坛、棋类 Discord、给中小chess YouTuber 送 lifetime code

## 11. 非目标(明确不做)

- 人机对弈 / 在线对弈(Chess.com 的地盘,不碰)

- 社区/评论功能(运营成本高,V1 不做)

## 12. 中国市场策略

经一轮调研后确立(2026 政策已变,务必按现行口径)。核心原则:**把"语言"和"市场/主体/平台"分开看。**

### 12.1 语言支持(中文)

- 中文是明确目标,不再推迟。技术上**复用 TECH_SPEC §5.1 的"先算事实后叙述"架构**:事实语言无关,叙述步加 `lang` 参数即可生成中文讲解(是"用中文叙述同一批事实",不是翻译英文 prose)。
- 配两张确定性表:战术母题→中文术语(牵制/闪击/双吃/弃子/兑子/将杀;注意"象车杀"非"双象杀"),棋手名→中译(费舍尔/卡尔森/塔尔)。术语来自 `facts.py`,不让 LLM 乱译。
- 从第一天起 i18n 干净(String Catalog,不硬编码),即便先发英文。
- 中文讲解质检需会中文棋术语的人(Jerry)。

### 12.2 三个市场层,分平台分主体

| 层 | 平台 / 渠道 | 主体 | 状态 |
|---|---|---|---|
| 海外(美/英 + 台/港/新 + 海外华人) | 原生 iOS app(App Store) | **个人开发者(Jerry)** | 近期并行轨,覆盖海外 |
| 中国大陆 · 主入口 | **微信小程序** | **中国公司(Jerry 已有一个收费小程序在上)** | 近期并行轨,基础设施已就绪 |
| 中国大陆 · 后补 | 原生 App Store(可选) | 中国公司 | 仅当重度订阅用户值得时 |

**重要前提(改变了排序):** Jerry 已有一个**收费小程序即将上线**,主体 / ICP / 微信支付与虚拟支付接入 / 备案 / 上架实操经验**都已付过**。所以小程序的边际成本塌到只剩"客户端开发 + 内容(共享)+ 单独推广"——它**不再是靠后阶段,而是近期并行轨**。继承的是基础设施,**不继承**用户/分发(推广仍要单独想办法)。

**关键洞察:主体冲突用"平台分工"绕过,不用解。** 同一 bundle ID 不能挂两个苹果账号,个人账号又因 ICP 上不了大陆——但"个人/海外原生 + 公司/中国小程序"天然分在两个平台,**根本不产生重复 app / 双账号问题**。

### 12.3 大陆为什么走小程序为主

- **分发是中国的胜负手**,小程序是小团队唯一能不靠大广告费拿到病毒传播的渠道;我们的"每日挑战分享卡"机制天然契合微信群/朋友圈,少儿教育家长群尤甚。
- **2026 政策变化**:苹果与腾讯达成协议,自 2026-02-27 起**微信小程序 iOS 端全面支持虚拟支付**(苹果抽 15%,开发者到手约 85%);带虚拟支付的小程序须 2026-04-01 前完成全端接入。"iOS 小程序不能卖数字内容"的旧约束已作废。
- 还避开**安卓多渠道地狱**(华为/小米/OPPO/vivo/应用宝各自上架备案),把整个中国收敛到一个微信入口。
- 弱点:留存弱于原生(用完即走、推送受限)、包体上限(主包 2MB)。对策:streak + 服务通知/公众号召回顶留存;内容(50+ 局)走境内备案域名按需拉取,不全打进包。

### 12.4 走"教育"类目,不走游戏

- 对标 **国象联盟**(北京棋易科技):带完整对弈都能归"教育"类目并收会员费,证明教育路线可行、绕开游戏版号长队。
- 我们**更占优**:无真人实时对弈(只是"猜大师着法"+AI 教学),"游戏"信号更弱、"教育"属性更纯,被判游戏/要版号的风险更低。
- ⚠️ **灰色点(上架前必须由办主体/代上架方按现行尺度确认)**:教育类目(尤其小程序)开虚拟支付**是否仍被要求版号**——NPPA/苹果对"带棋类玩法的教育 app"口径有灰色地带且会变。

### 12.5 大陆合规清单(有主体后办,都可做)

- ICP备案(主体 + 境内云 + 域名备案)
- 教育App备案(教育移动应用备案,省级教育部门)
- 数据合规(PIPL:内容存境内、中文隐私政策、SDK 报备)
- 我们的**无后端 + 静态内容设计让合规面极小**:内容托管到境内备案域名即可;无 UGC/无聊天/无实时对弈,内容审核与防沉迷负担都轻。

### 12.6 节奏(共享地基先行,两端近乎并行)

排序已按"Jerry 已有收费小程序、中国基础设施就绪"修正——小程序不再靠后。

1. **先建共享地基**:pipeline + 数据契约(`/shared` schema + 黄金测试向量)+ 薄客户端架构(逻辑预计算进数据,见 TECH_SPEC §5.1 / AGENTS)。两个客户端的共同依赖,**最先做、最该做扎实**;两端都近在眼前,这套"把胖逻辑留在数据层"的决定更紧迫,不是更不急。
2. **核心循环验证一次**(手感 + 讲解质量)——从**任一**客户端出即可;鉴于中国支付/合规链路已就绪,**小程序很可能是更快够到真实付费用户的那条**,验证可以在国内市场里先发生。
3. **两端近乎并行推进**:原生 app 覆盖海外(个人主体),小程序覆盖中国(公司主体,复用同一份流水线和数据)。谁先够到付费用户谁先冲,不互相阻塞。
4. **大陆原生 App Store** 仍是后补的"盾":小程序跑出量且重度订阅用户明显时再上(独立 bundle ID + 中国公司账号 + 区域不重叠,海外版排除大陆区),服务高端段。

一句话:**先把共享地基打牢,海外原生与中国小程序近乎并行;小程序因基础设施已就绪,可以很早上,甚至是中国最快见付费的一条。** 仍要守住的唯一先后:核心循环先验证一次,再全力铺两端,别在概念未验证前就分散精力做两套完整客户端。
