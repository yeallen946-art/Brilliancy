## 1. 导航结构

底部 TabBar,三个 tab(V1 克制,不超过三个):

```
┌──────────────┬──────────────┬──────────────┐
│   Today      │   Train      │   Progress   │
│  每日挑战     │   训练库      │   进步追踪    │
└──────────────┴──────────────┴──────────────┘
```

- **Today**(默认 tab):每日挑战入口,免费用户也能完整玩
- **Train**:题包浏览,付费墙在此
- **Progress**:棋感评分曲线、弱点分析(付费)

Settings 不占 tab,放在各 tab 右上角的齿轮入口(或 Progress 顶部)。
GuessSession(猜着法循环)是模态全屏,从 Today 或 Train 推入,不属于任何 tab。

### S2 — GuessSession(核心循环,模态全屏)
这是产品的心脏。按状态分帧描述(状态机见 TECH_SPEC §3.1):

```
[SHOW_CONTEXT] 进局首帧
┌─────────────────────────────────┐
│  ✕                      Game 1/1 │
│  Byrne vs Fischer · 1956 · NY    │
│  You play Black (Fischer).        │  ← 明确告知执子方
│         [棋盘:起始局面]           │
│            [ Begin ]              │
└─────────────────────────────────┘

[WALK_IN] 点 Begin 后,棋盘从起始局面**一步一步快速走到第一个猜测点**
(~250-400ms/步,带走子动画),绝不直接跳到局面。让玩家跟上对局脉络。
每个猜测点之间的过渡同理(走对手应着 + 跳过段),都是逐步动画,不是瞬切。

[AWAIT_GUESS] 等待用户走子
┌─────────────────────────────────┐
│  ✕              Move 14 · 🔥 streak│
│                                   │
│        [ 交互棋盘 ]               │  ← 拖拽或点-点走子
│                                   │
│  Black to move. What did Fischer  │
│  play?                            │
│  [Hint 💡(付费/限量)]            │  ← 可选,V1 可后置
└─────────────────────────────────┘

[SCORE → REVEAL] 走子后揭示
┌─────────────────────────────────┐
│  ✓ Match!  / ≈ Equally strong /   │  ← 三档反馈,色带(绿/黄/红)
│  ✗ Inaccuracy (-0.8)              │
│                                   │
│   [棋盘:高亮你的着法 vs 大师着法]  │
│                                   │
│  ┌─ AI 讲解(核心差异化) ──────┐ │
│  │ Fischer played Be6!! — a quiet │ │
│  │ move that ignores the queen... │ │
│  │ Your Nxd4 loses to ... because │ │  ← 命中也讲"为什么好";错了讲"为什么不如它"
│  │ ...                            │ │
│  └────────────────────────────────┘ │
│            [ Next → ]             │
└─────────────────────────────────┘

[WALK_TO_NEXT] 对手应着 + 中间着法**逐步走子动画**(~250-400ms/步)走到下一个猜点,绝不瞬切
```
关键体验:讲解区是主角,给足空间;反馈语气是教练不是判官(✗ 也不用红得刺眼地羞辱)。

## 4. 视觉约束(方向,非高保真)

不画精细稿,但定下这些约束让 agent 第一版不跑偏:

- **整体调性:** 克制、专注、像训练工具不像游戏厅。深色为主(降低长时间看棋盘的疲劳),浅色可选。
- **棋盘:** 遵循既定范式——经典绿白 / 木纹两套起步,可换肤但 V1 只做 2-3 套。
- **棋子(硬性可辨识要求,V1 必须达标):** 棋子是产品的核心交互对象,**易于分辨优先于风格**。
  - 选一套高对比、轮廓清晰、棋子之间形状区分度大的成熟矢量集(如 Lichess 的 cburnett/merida,注意许可)。**避免**线条纤细、各子剪影相近、低对比的棋子集。
  - 白子与深色格、黑子与浅色格都要有足够对比;黑白子之间必须有明确的描边/填充差异,不能只靠明暗。
  - 棋子要占满格子的合理比例(不要太小),在 iPhone 最小屏宽下单子仍清晰可辨。
  - 验收:在真机上扫一眼能立刻区分马/象、王/后——分不清就是没达标。这是 bug 级要求,不是品味问题。
- **强调色:** 一个主品牌色用于 CTA 和 streak;反馈三色固定——绿(命中)、黄(尚可)、红(失误),红要克制不刺眼。
- **字体:** 系统字体(SF Pro)起步,讲解正文可读性优先,行高放宽。
- **语气(文案):** 教练而非判官。错误反馈用"This loses material because..."而非"Wrong!"。PRD §5 原则的 UI 落地。
- **动效:** 走子有平滑动画;揭示时大师着法的高亮要清楚。**进入猜测点永远逐步走子动画到位,绝不瞬切跳到局面**(走入/过渡 ~250-400ms/步,见 TECH_SPEC §3.1)。不要花哨转场。
- **可达性:** Dynamic Type 支持(讲解会被放大字号的用户读);色盲友好——三档反馈除颜色外用图标(✓ ≈ ✗)区分,不单靠颜色。

### 4.1 视觉设计系统(M1 实现标准:Code 照此实现,Jerry 照此验收)

经多轮 mockup 迭代确定的具体规格。hex 为方向锚点,M1 可在模拟器微调,但"棋子可辨识 + Chess.com 绿白盘 + 深色外壳 + 金色 CTA"这套关系不变。

**配色(深色主题):**

- 背景 `#0F141A`;卡片表面 `#1B222B`;描边 `#28313B`
- 正文字 `#ECF0F4`;次要字 `#94A1AE`
- 品牌金(仅用于主 CTA 和 streak)`#E6B450`;金底上的深字 `#2E2103`
- 反馈三色(均调柔,红不刺眼):命中绿 `#57B98A`、尚可黄 `#E0B84A`、失误红 `#E07A6A`

**棋盘(Chess.com 默认绿白风格):**

- 浅格 `#EBECD0`,深格 `#769656`
- 高亮(上一步 / 大师着法的起止格):整格黄色叠加,浅格 `#F5F682`、深格 `#B9CA43`(整格着色,不是细描边环)

**棋子(选定:cburnett):**

- 棋子集 **cburnett**(作者 Colin M.L. Burnett,CC BY-SA 3.0,可商用;须在 app"关于/致谢"注明出处)。备选 merida、cardinal。
- **明确不用 Chess.com 的 neo 等专有棋子集**(侵权);只参考其盘面配色与棋子尺寸感,不照搬美术资产。
- **SVG 资产打包进 app 本地**,不热链 CDN(离线可用、稳定、合规)。
- 尺寸按各子自然比例渲染、占满格子(大子约 90%,兵约 65%),不要统一拉大;浅子近白 + 深描边,深子近黑 + 细浅描边/光晕(保证在深绿格上轮廓清晰)。

**间距 / 圆角 / 字号:**

- 间距用 8 的倍数(8 / 12 / 16 / 24);卡片圆角 12、按钮 12、棋盘 4–6
- 字号层级:对局信息 13、讲解正文 13(行高 1.6)、小标签 11(大写带字距)
- 仅两种字重:常规 400、半粗 500,不用更重

**揭示页(S2)视觉动线(从上到下):** 顶栏(关闭 / 对局+步数 / streak)→ 棋盘(高亮着法起止格)→ 反馈条 → 讲解卡(核心差异化,给足空间)→ 金色主按钮。

## 5. 哪些现在定、哪些 code 里定
